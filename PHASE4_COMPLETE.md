## Lessons & Best Practices

### Architecture
- **Scraper Abstraction**: BaseScraper pattern enables unlimited store scaling
- **Dynamic Configuration**: Store configs in database, no hardcoded URLs
- **Service Layer**: Clear separation between routes, services, scrapers
- **Error Resilience**: One store failure doesn't block others

### Deployment
- **Infrastructure as Code**: Rails provides version control
- **Environment Separation**: .env for local, .env.production for prod
- **Monitoring First**: Logs available before issues occur
- **Automation**: Deploy with single `git push`

### Operations
- **Daily Schedules**: APScheduler handles recurring jobs
- **Rate Limiting**: 3-sec delays prevent IP blocking
- **Data Deduplication**: Removes duplicates across stores
- **Graceful Degradation**: System works even if some stores fail
