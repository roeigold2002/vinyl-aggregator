# Phase 4: Production Deployment & Live Monitoring

## Overview

Phase 4 is the final phase where the Israeli Vinyl Record Aggregator moves from development to production:

1. **GitHub Repository** - Push complete codebase to GitHub
2. **Backend Deployment** - Deploy to Railway with PostgreSQL
3. **Frontend Deployment** - Deploy to Vercel
4. **Production Initialization** - Set up database and seed stores
5. **Scraper Activation** - Trigger first production scrape
6. **Monitoring Setup** - Configure logs, alerts, and dashboards
7. **Production Verification** - Test all endpoints and functionality

**Timeline**: 2-3 hours from start to live system

## Step 2: Deploy Backend to Railway

### 2.1 Railway Project Setup

```bash
# Install Railway CLI
npm install -g @railway/cli

# Login to Railway
railway login

# Navigate to backend directory
cd backend

# Initialize Railway project
railway init

# This will prompt you to:
# 1. Create new project
# 2. Add PostgreSQL database
# 3. Confirm environment variables
```

### 2.2 Configure Railway Environment Variables

In Railway dashboard (https://railway.app):

1. Go to Project → Settings → Variables
2. Add variables:

```
DATABASE_URL             [Auto-generated by PostgreSQL plugin]
REDIS_URL               [Optional, skip if not using]
ENVIRONMENT             production
LOG_LEVEL               INFO
CORS_ORIGINS            ["https://your-vercel-frontend.vercel.app"]
```

### 2.3 Deploy Backend

```bash
# Deploy to production
railway up

# View logs
railway logs -f

# Get production URL
railway env
```

**Expected logs**:
```
✅ FastAPI server running on 0.0.0.0:8000
✅ CORS enabled for specified origins
✅ Database connected
✅ Scheduler initialized
```

**Success**: Backend URL looks like `https://your-app.up.railway.app`

## Step 4: Initialize Production Database

### 4.1 Connect to Production Database

```bash
# Get database URL from Railway
railway env | grep DATABASE_URL

# Export it
export DATABASE_URL="postgresql://..."
```

### 4.2 Run Database Initialization

```bash
cd backend

# Initialize schema
python << 'EOF'
from database import init_db, SessionLocal
db = SessionLocal()
init_db(db)
print("✅ Database schema initialized")
db.close()
EOF

# Seed stores
python << 'EOF'
from database import SessionLocal, seed_stores
db = SessionLocal()
seed_stores(db)
print("✅ 10 stores seeded into database")
db.close()
EOF

# Verify
python << 'EOF'
from database import SessionLocal
from models import Store
db = SessionLocal()
stores = db.query(Store).all()
print(f"✅ Stores in database: {len(stores)}")
for store in stores:
    print(f"   - {store.name}")
db.close()
EOF
```

**Expected output**:
```
✅ Database schema initialized
✅ 10 stores seeded into database
✅ Stores in database: 10

   - The Third Ear
   - Beatnik
   - Giora Records
   - Has Ivoov
   - Shablool Records
   - Vinyl Stock

   - Takli House
```

```bash
# Check slow queries
SELECT query, mean_time

SELECT datname, count(*) FROM pg_stat_activity GROUP BY datname;
```

- Disabled job runs all 12 scrapers in sequence

### 8.3 Database Maintenance

```bash
# Weekly optimization

VACUUM ANALYZE;
REINDEX INDEX CONCURRENTLY idx_records_artist;

SELECT COUNT(*) FROM records;
SELECT COUNT(DISTINCT artist, album) FROM records;
```

## Troubleshooting Production Issues

### Issue: Backend deployment fails

```bash
# Check logs
railway logs

# Verify environment variables
railway env

# Check Python version
python --version  # Should be 3.11+

# Redeploy
railway down
railway up
```

### Issue: Frontend can't connect to backend

```bash
# Verify backend URL in .env
cat frontend/.env.production

# Check CORS settings in backend
# Should include frontend URL

# Redeploy frontend if needed
cd frontend
vercel --prod
```

### Issue: Scraper job not running

```bash
# Check if in database
psql $DATABASE_URL -c "
SELECT * FROM apscheduler_locks;
SELECT * FROM apscheduler_jobs;
"

# Check logs for scheduler errors
railway logs | grep "scheduler"

# Manually trigger
curl -X POST "https://your-app.up.railway.app/api/stores/scrape"
```

### Issue: Search returns 0 results

```bash
# Check database has data
psql $DATABASE_URL -c "SELECT COUNT(*) FROM records;"

# Rebuild search index if needed
psql $DATABASE_URL -c "
UPDATE records SET search_vector = 
  to_tsvector('english', artist || ' ' || album || ' ' || title);
DELETE FROM gin_pending_list('idx_records_search');
"
```
